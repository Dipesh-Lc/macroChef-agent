# Limitations

- MacroChef Agent is not medical advice.
- Nutrition estimates depend on recipe metadata and may not match packaged food labels.
- Vision extraction may be uncertain; users should confirm detected ingredients.
- Allergy safety depends on accurate recipe metadata and accurate user input.
- The bundled recipe dataset is intentionally small for a portfolio MVP.
- Mock vision and mock explanations are deterministic by default so the app runs without paid API keys.
- Hosted/local model integrations are optional and should not be treated as allergy or nutrition authorities.
- Real grocery checkout is out of scope for the MVP.
